Reusable snippets help you maintain consistency across your documentation by letting you write content once and reuse it in multiple places. When you update a snippet, all pages using it automatically reflect the changes.

<Note>
  Any page in the `snippets` directory is treated as a snippet and will not be rendered as a standalone page.
</Note>

## Creating snippets

### Basic setup

Create a `snippets` directory in your docs root if it doesn't exist:

```bash
mkdir snippets
```

All snippet files must live in this directory:

```
docs/
├── snippets/
│   ├── prerequisites.mdx
│   ├── installation.mdx
│   └── support-info.mdx
└── getting-started.mdx
```

## Default export snippets

The simplest way to create a reusable snippet.

### Creating the snippet

Create a file in the `snippets` directory:

```mdx snippets/api-key-note.mdx
<Warning>
  Keep your API key secure! Never commit it to version control or share it publicly.
</Warning>

You can find your API key in the [dashboard](https://dashboard.example.com/settings/api).
```

### Using the snippet

Import and use the snippet in any page:

```mdx getting-started.mdx
---
title: Getting started
description: Learn how to get started
---

import ApiKeyNote from '/snippets/api-key-note.mdx';

## Authentication

Our API uses API keys for authentication.

<ApiKeyNote />

## Making requests

Include your API key in the Authorization header...
```

## Snippets with variables

Make snippets dynamic by accepting props:

### Creating the snippet

**File: snippets/installation-guide.mdx**
```mdx
Install {props.packageName} using your preferred package manager.

Use CodeGroup with package manager tabs and pass packageName as a prop.

Conditionally show a note if props.note is provided.
```

### Using the snippet with props

```mdx my-page.mdx
---
title: Installation
---

import InstallationGuide from '/snippets/installation-guide.mdx';

## Installing the SDK

<InstallationGuide 
  packageName="@example/sdk" 
  note="Requires Node.js 16 or higher"
/>

## Installing the CLI

<InstallationGuide 
  packageName="@example/cli"
  note="Add -g flag for global installation"
/>
```

## Reusable variables

Export variables from snippets to reuse across pages:

### Defining variables

```mdx snippets/constants.mdx
export const companyName = 'Acme Corp';
export const apiVersion = 'v2';
export const supportEmail = 'support@example.com';

export const endpoints = {
  production: 'https://api.example.com',
  staging: 'https://staging-api.example.com',
  development: 'http://localhost:3000'
};

export const pricing = {
  starter: '$29/month',
  professional: '$99/month',
  enterprise: 'Custom'
};
```

### Using variables

```mdx pricing-page.mdx
---
title: Pricing
---

import { companyName, pricing } from '/snippets/constants.mdx';

# {companyName} Pricing

Choose the plan that works for you:

- **Starter**: {pricing.starter}
- **Professional**: {pricing.professional}
- **Enterprise**: {pricing.enterprise}
```

```mdx api-reference.mdx
---
title: API Reference
---

import { apiVersion, endpoints } from '/snippets/constants.mdx';

## Base URL

The base URL for the {apiVersion} API is:

```
{endpoints.production}
```

For testing, use the staging environment:

```
{endpoints.staging}
```
```

## Reusable components

Create complex reusable components with custom logic:

### Creating a component

```mdx snippets/api-endpoint.mdx
export const ApiEndpoint = ({ method, path, description }) => {
  const methodColors = {
    GET: '#10b981',
    POST: '#3b82f6',
    PUT: '#f59e0b',
    DELETE: '#ef4444',
    PATCH: '#8b5cf6'
  };
  
  return (
    <div style={{
      border: '1px solid #e5e7eb',
      borderRadius: '0.5rem',
      padding: '1rem',
      marginBottom: '1rem'
    }}>
      <div style={{ display: 'flex', alignItems: 'center', gap: '1rem' }}>
        <span style={{
          backgroundColor: methodColors[method],
          color: 'white',
          padding: '0.25rem 0.75rem',
          borderRadius: '0.25rem',
          fontWeight: 'bold',
          fontSize: '0.875rem'
        }}>
          {method}
        </span>
        <code style={{ fontSize: '1rem' }}>{path}</code>
      </div>
      {description && (
        <p style={{ marginTop: '0.5rem', marginBottom: 0 }}>{description}</p>
      )}
    </div>
  );
};
```

<Warning>
  MDX does not compile inside arrow function components. Use HTML syntax instead of MDX syntax in component bodies.
</Warning>

### Using the component

```mdx api-docs.mdx
---
title: API Endpoints
---

import { ApiEndpoint } from '/snippets/api-endpoint.mdx';

## User Management

<ApiEndpoint 
  method="GET" 
  path="/api/users" 
  description="Retrieve a list of all users"
/>

<ApiEndpoint 
  method="POST" 
  path="/api/users" 
  description="Create a new user"
/>

<ApiEndpoint 
  method="DELETE" 
  path="/api/users/:id" 
  description="Delete a user by ID"
/>
```

## Best practices

<AccordionGroup>
  <Accordion title="Keep snippets focused" icon="bullseye">
    Create snippets for specific, reusable pieces of content:
    
    ✅ Good: `installation-steps.mdx`, `api-authentication.mdx`
    
    ❌ Bad: `everything.mdx`, `misc-content.mdx`
  </Accordion>
  
  <Accordion title="Organize by category" icon="folder-tree">
    Structure your snippets directory:
    
    ```
    snippets/
    ├── api/
    │   ├── authentication.mdx
    │   ├── rate-limits.mdx
    │   └── errors.mdx
    ├── installation/
    │   ├── npm.mdx
    │   ├── prerequisites.mdx
    │   └── troubleshooting.mdx
    └── common/
        ├── support.mdx
        └── legal.mdx
    ```
  </Accordion>
  
  <Accordion title="Document snippet props" icon="file-lines">
    Add comments to document expected props:
    
    ```mdx snippets/installation.mdx
    {/* 
      Props:
      - packageName (string, required): The npm package name
      - minVersion (string, optional): Minimum required version
      - note (string, optional): Additional installation note
    */}
    
    Install {props.packageName}...
    ```
  </Accordion>
  
  <Accordion title="Use variables for values that change" icon="rotate">
    Extract frequently changing values:
    
    ```mdx snippets/versions.mdx
    export const currentVersion = '2.1.0';
    export const minNodeVersion = '16.0.0';
    export const releaseDate = 'March 2026';
    ```
    
    This makes updates easier and keeps documentation consistent.
  </Accordion>
  
  <Accordion title="Test snippets after changes" icon="vial">
    When you update a snippet:
    
    1. Check all pages that use it
    2. Run `mint broken-links` to catch any issues
    3. Preview in development mode
    4. Look for any formatting or content issues
  </Accordion>
</AccordionGroup>

## Advanced patterns

### Conditional content

Show different content based on props:

```mdx snippets/platform-guide.mdx
export const PlatformGuide = ({ platform }) => {
  const guides = {
    macos: (
      <div>
        <h3>macOS Installation</h3>
        <p>Use Homebrew: <code>brew install example</code></p>
      </div>
    ),
    windows: (
      <div>
        <h3>Windows Installation</h3>
        <p>Download the installer from our website</p>
      </div>
    ),
    linux: (
      <div>
        <h3>Linux Installation</h3>
        <p>Use your package manager: <code>apt install example</code></p>
      </div>
    )
  };
  
  return guides[platform] || <p>Platform not supported</p>;
};
```

### Nested snippets

Snippets can import other snippets:

```mdx snippets/full-installation.mdx
import Prerequisites from '/snippets/prerequisites.mdx';
import InstallationSteps from '/snippets/installation-steps.mdx';
import Verification from '/snippets/verification.mdx';

<Prerequisites />
<InstallationSteps />
<Verification />
```

## Troubleshooting

<Warning>
  If your snippet isn't working, check:
  
  - The file is in the `snippets` directory
  - The import path starts with `/snippets/`
  - The file extension is `.mdx`
  - Component names are capitalized (PascalCase)
</Warning>

### Common issues

**Import not found**
```
Error: Cannot find module '/snippets/my-snippet.mdx'
```
Solution: Ensure the path is correct and starts with `/snippets/`

**Component not rendering**
```
Nothing appears when I use the snippet
```
Solution: Make sure you're using JSX syntax: `<MySnippet />` not `{MySnippet}`

**MDX not compiling in component**
```
MDX syntax doesn't work in my arrow function
```
Solution: Use HTML syntax inside arrow functions, or use default exports for MDX content
